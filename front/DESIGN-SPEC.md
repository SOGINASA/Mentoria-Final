# DESIGN-SPEC — описание экранов для дизайна

Документ для передачи в **Claude Design**: что должно быть на каждом экране согласно ТЗ.
Стек реализации — React + Tailwind + Zustand. **Приоритет — мобильная версия**, десктоп — адаптация.

---

## 0. Общие правила дизайна

- **Mobile-first.** Базовый макет — телефон (360–430px). Десктоп — расширение того же контента
  (контентная колонка по центру / двухколоночные раскладки на широких экранах).
- **Целевой пользователь** — сотрудник на торговой точке, часто в спешке, одной рукой, на ходу.
  Крупные тач-цели (≥44px), минимум полей на экране, понятные подписи, мало текста.
- **Навигация:**
  - Мобайл — нижний таб-бар (`BottomNav`) с 3–4 пунктами в зависимости от роли.
  - Десктоп — боковое меню (`Sidebar`) + верхний `Header`.
- **Тон:** деловой, чистый, доверительный. Никаких «игровых» элементов — это рабочий инструмент.
- **Статусы заявок** — единая цветовая система во всём приложении:
  - 🟡 На проверке (PENDING)
  - 🟢 Подтверждена (APPROVED)
  - 🔴 Отклонена (REJECTED)
- **Состояния экранов:** для каждого списка/формы предусмотреть: загрузка (skeleton/spinner),
  пусто (EmptyState), ошибка, успех (toast).
- **Фото** — ключевой элемент (подтверждение фактического состояния продукции).
  Делать заметным, с превью и возможностью пересъёмки.

## Роли и их экраны

| Роль | Экраны |
|------|--------|
| **Отправитель** (сотрудник точки) | Вход → Главная → Создание заявки → Мои заявки → Деталь заявки → Профиль |
| **Проверяющий** | Вход → Очередь проверки → Проверка заявки → История → Профиль |

---

## 1. Вход (LoginPage) — `/login`

**Цель:** индивидуальный доступ для каждого пользователя (требование ТЗ).

**Содержимое:**
- Логотип / название системы.
- Поля: логин (или телефон/email) + пароль.
- Кнопка «Войти» (основная, во всю ширину).
- Состояние ошибки авторизации (неверные данные).
- Опционально: «Запомнить меня».

**Макет:**
- Мобайл — вертикально по центру, поля во всю ширину, кнопка крупная снизу.
- Десктоп — центрированная карточка (макс. ~400px), фон/иллюстрация по бокам.

**После входа:** роль определяет стартовый экран (отправитель → Главная, проверяющий → Очередь).

---

## 2. Главная отправителя (SenderHomePage) — `/`

**Цель:** быстрый старт новой заявки + обзор своих последних заявок.

**Содержимое:**
- Приветствие / имя пользователя, его торговая точка (если закреплена).
- **Крупная главная кнопка «➕ Создать списание»** (основное действие, заметная,
  «липкая» внизу на мобиле или большая карточка-кнопка вверху).
- Краткая сводка: счётчики «На проверке / Подтверждено / Отклонено» за период.
- Список последних 3–5 заявок (карточки со статусом) + ссылка «Все мои заявки».

**Макет:**
- Мобайл — одна колонка: сводка → кнопка → последние заявки.
- Десктоп — две колонки: слева сводка+кнопка, справа список последних заявок.

---

## 3. Создание заявки (CreateWriteOffPage) — `/create`

**Цель:** отправитель оформляет списание. Это **ключевой экран приложения.**
Лучше всего — **пошаговый мастер** (1 действие на шаг → удобно на телефоне), с прогресс-индикатором.
На десктопе шаги можно показать единой формой в колонку.

Поля строго по ТЗ (порядок процесса):

**Шаг 1. Фотография продукции** *(обязательно)*
- Кнопка «Сделать фото» (камера) + «Загрузить из галереи».
- Превью снимка, возможность удалить/переснять.
- Можно несколько фото (минимум одно).

**Шаг 2. Торговая точка** *(обязательно)*
- Выбор из списка точек (`Select` / выпадающий список с поиском, если точек много).
- Если за пользователем закреплена одна точка — предзаполнить.

**Шаг 3. Тип списания** *(обязательно)*
- Переключатель из двух вариантов (крупные карточки-радио):
  - **Без удержания с сотрудника**
  - **С удержанием с сотрудника**

**Шаг 4. Сотрудник для удержания** *(условно — только если выбран тип «с удержанием»)*
- Появляется/раскрывается только при «с удержанием».
- Выбор сотрудника из списка (`Select` с поиском).

**Шаг 5. Комментарий** *(обязательно, минимум 10 символов)*
- `Textarea` с подсказкой и **счётчиком символов**, валидация «минимум 10 символов»
  (кнопка отправки заблокирована, пока не выполнено).

**Действие:**
- Кнопка «Отправить на проверку» (активна только при валидной форме).
- После отправки — экран успеха («Заявка отправлена на проверку») с переходом в «Мои заявки».

**Валидация (показывать понятно):** фото есть, точка выбрана, тип выбран, при удержании —
сотрудник выбран, комментарий ≥ 10 символов.

**Макет:**
- Мобайл — один шаг на экран, прогресс сверху, кнопки «Назад/Далее» снизу, «липкая» нижняя кнопка.
- Десктоп — вся форма одной колонкой по центру (макс. ~600px), фото-блок крупный сверху.

---

## 4. Мои заявки / история (MyRequestsPage) — `/my-requests`

**Цель:** отправитель видит все свои заявки и их статусы (требование ТЗ — история списаний).

**Содержимое:**
- Фильтр/табы по статусу: Все / На проверке / Подтверждены / Отклонены.
- Поиск/фильтр по дате и точке (опционально).
- Список карточек заявок: миниатюра фото, тип списания, точка, дата, **StatusBadge**.
- Пусто → EmptyState («Заявок пока нет» + кнопка «Создать списание»).

**Макет:**
- Мобайл — вертикальный список карточек, табы статусов сверху (скролл-чипсы).
- Десктоп — таблица или сетка карточек, фильтры в боковой/верхней панели.

---

## 5. Деталь своей заявки (RequestDetailPage) — `/my-requests/:id`

**Цель:** отправитель смотрит полную информацию и решение по заявке.

**Содержимое:**
- Крупное фото (с возможностью открыть на весь экран).
- Все данные: точка, тип списания, сотрудник для удержания (если есть), комментарий, дата.
- **StatusBadge** + если отклонено — причина отклонения от проверяющего.
- Таймлайн: создана → проверена (кем, когда) → (для подтверждённых) акт создан в iiko.

**Макет:** мобайл — одна колонка; десктоп — фото слева, данные справа.

---

## 6. Очередь проверки (ReviewQueuePage) — `/review`

**Цель (роль проверяющего):** видеть входящие заявки на проверку.

**Содержимое:**
- Табы/фильтр: На проверке (по умолчанию) / Все / по точкам / по дате.
- Счётчик заявок, ожидающих проверки (бейдж).
- Список карточек: миниатюра фото, автор (сотрудник), точка, тип (с/без удержания),
  сотрудник для удержания, дата, **StatusBadge**.
- Тап по карточке → экран проверки.
- Пусто → EmptyState («Нет заявок на проверку»).

**Макет:**
- Мобайл — список карточек, фильтры — скролл-чипсы сверху.
- Десктоп — таблица с колонками + быстрый просмотр; возможен сплит «список слева / деталь справа».

---

## 7. Проверка заявки (ReviewDetailPage) — `/review/:id`

**Цель:** проверяющий изучает фото и данные, **подтверждает или отклоняет** (ключевой экран роли).

**Содержимое:**
- **Крупное фото** продукции (зум / на весь экран) — главный элемент проверки.
- Полные данные заявки: автор, точка, тип списания, сотрудник для удержания (если есть),
  комментарий, дата/время.
- **Два явных действия:**
  - 🟢 **Подтвердить** — подтверждение в модалке; после — запускается создание акта в iiko.
  - 🔴 **Отклонить** — в модалке обязательная причина отклонения (вернётся отправителю).
- Индикация интеграции iiko: после подтверждения — статус «Акт списания создаётся/создан в iiko».

**Макет:**
- Мобайл — фото сверху, данные ниже, **«липкая» нижняя панель с двумя кнопками** (Подтвердить/Отклонить).
- Десктоп — фото слева (большое), данные и кнопки справа.

---

## 8. История проверок (ReviewHistoryPage) — `/review/history`

**Цель:** проверяющий видит обработанные заявки (история всех списаний — требование ТЗ).

**Содержимое:**
- Фильтр по статусу (Подтверждены / Отклонены / Все), по дате, по точке, по сотруднику.
- Список/таблица обработанных заявок со статусом и решением.
- Возможность открыть деталь (read-only).

**Макет:** мобайл — список карточек; десктоп — таблица с фильтрами и, опционально, экспортом.

---

## 9. Профиль (ProfilePage) — `/profile`

**Содержимое:**
- Имя, роль, торговая точка(и).
- Настройки (язык/уведомления — опционально).
- Кнопка «Выйти».

**Макет:** простая колонка на мобайле; карточка по центру на десктопе.

---

## 10. Общие компоненты (для консистентности дизайна)

- **AppShell** — каркас: контент + навигация. Мобайл: контент + нижний таб-бар.
  Десктоп: боковое меню + верхний хедер + контент.
- **BottomNav** (мобайл) — пункты по роли:
  - Отправитель: Главная · Создать · Мои заявки · Профиль
  - Проверяющий: Очередь · История · Профиль
- **Sidebar** (десктоп) — те же пункты, вертикально, с подписями и иконками.
- **StatusBadge** — единые цвета статусов (см. раздел 0).
- **Card** — карточка заявки в списках (фото-миниатюра, ключевые поля, статус).
- **PhotoUpload** — съёмка/загрузка + превью + удаление.
- **Modal** — подтверждение действий (одобрить/отклонить с причиной).
- **EmptyState / Spinner / Toast** — для пустых, загрузочных и результатных состояний.
