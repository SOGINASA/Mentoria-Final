# Back — сервер

Серверная часть системы списаний. Заполняется отдельно.

## Зона ответственности

- Аутентификация и индивидуальный доступ пользователей (роли: отправитель, проверяющий).
- CRUD заявок на списание + хранение фотографий.
- Очередь проверки заявок, подтверждение / отклонение.
- История всех списаний.
- Интеграция с **iiko**: после подтверждения — автоматическое создание акта
  списания и передача данных (списание остатков).

## Предполагаемые сущности

- `User` — id, ФИО, роль, торговые точки.
- `Store` — торговая точка.
- `WriteOff` (заявка) — фото, точка, тип (с удержанием / без), сотрудник-должник,
  комментарий, статус (на проверке / подтверждена / отклонена), автор, даты.
- `IikoActLink` — связь подтверждённой заявки с актом в iiko.
